## Contexto adicional

### Flujo generate-daily-summary

```
POST { business_id, date, type: 'daily' | 'first_week' }
  → cargar business (name, whatsapp_phone_number_id, whatsapp_access_token, timezone)
  → cargar owner (phone_personal)
  → agregar conversations WHERE business_id = X AND last_message_at BETWEEN date_start AND date_end
  → si total === 0 → return { skipped: true }
  → calcular métricas
  → detectar weak_topics (knowledge_count = 0)
  → buildDailySummaryMessage() → texto en español
  → sendWhatsAppMessage() → Meta API (best-effort)
  → upsert daily_summaries (onConflict: business_id, date)
  → return { success, summary_id, total_conversations, whatsapp_sent, whatsapp_error }
```

### Bug corregido en sesión

**Root cause:** `generate-daily-summary` y `intelligence/page.tsx` filtraban por `conversations.created_at`.
Una conversación con `created_at = 2026-03-15` y `last_message_at = 2026-03-20` NO aparecía en el resumen del día 20, aunque tenía actividad hoy.
**Fix:** `.gte('last_message_at', dateStart).lte('last_message_at', dateEnd)`.
